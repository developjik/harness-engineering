# Brainstorming Skill

## 개요

사용자의 요구사항을 깊이 있게 분석하고, 다양한 설계 아이디어를 도출하는 스킬입니다.

## 목표

- 사용자의 진정한 의도 파악
- 기술적 트레이드오프 분석
- 여러 설계 옵션 제시
- 최적의 접근 방식 추천

## 프로세스

### 1단계: 요구사항 정제 (Clarification)
사용자의 초기 요구사항을 명확히 합니다.

**질문 예시:**
- 이 프로젝트의 주요 목표는 무엇인가요?
- 성공의 기준은 무엇인가요?
- 제약 조건이 있나요? (시간, 리소스, 기술 스택)
- 기존 시스템과의 통합이 필요한가요?

### 2단계: 아이디어 도출 (Ideation)
여러 설계 옵션을 제시합니다.

**고려 사항:**
- 아키텍처 패턴 (MVC, MVVM, Clean Architecture 등)
- 기술 스택 선택
- 확장성 및 유지보수성
- 성능 및 보안

### 3단계: 평가 및 선택 (Evaluation)
각 옵션의 장단점을 분석합니다.

**평가 기준:**
- 복잡도
- 개발 시간
- 유지보수 난이도
- 확장 가능성
- 팀의 경험도

### 4단계: 설계 문서 작성 (Documentation)
선택된 설계를 문서화합니다.

## 산출물

```
design-document.md
├── 요구사항 정리
├── 제약 조건
├── 아키텍처 다이어그램
├── 기술 스택 선택 이유
├── 위험 요소 및 완화 전략
└── 구현 단계별 계획
```

## 체크리스트

- [ ] 사용자의 모든 요구사항 이해
- [ ] 최소 3가지 설계 옵션 제시
- [ ] 각 옵션의 장단점 분석
- [ ] 선택된 설계에 대한 명확한 이유
- [ ] 구현 가능성 검증
- [ ] 설계 문서 작성 완료
- [ ] 사용자 승인 획득

## 예제

### 입력
```
"사용자 인증 기능이 있는 웹 애플리케이션을 만들고 싶어요."
```

### 프로세스
1. 요구사항 정제: 어떤 종류의 인증? (로컬, OAuth, SAML?) 몇 명의 사용자?
2. 아이디어 도출: JWT + Express, Firebase Auth, Auth0 등
3. 평가: 팀의 경험도, 비용, 보안 요구사항 고려
4. 문서화: 선택된 기술 스택 및 구현 계획 작성

### 산출물
```markdown
# 사용자 인증 시스템 설계

## 요구사항
- 로컬 사용자 인증 지원
- JWT 기반 토큰 관리
- 안전한 비밀번호 저장

## 선택된 기술
- Backend: Node.js + Express
- Database: PostgreSQL
- Authentication: JWT + bcrypt

## 구현 계획
1. 사용자 모델 설계
2. 인증 엔드포인트 구현
3. 토큰 관리 시스템
4. 테스트 작성
```

## 주의사항

- 사용자의 의견을 충분히 반영하세요
- 너무 복잡한 설계는 피하세요 (YAGNI)
- 팀의 경험도를 고려하세요
- 나중에 변경 가능한 결정과 불가능한 결정을 구분하세요
