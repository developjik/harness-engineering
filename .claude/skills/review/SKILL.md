# Review Skill

구현된 코드를 다각적으로 검토하여 품질을 보증합니다.

## 리뷰 기준

### 1. 기능 정확성 (Correctness)

**확인 사항:**
- 설계 문서와 일치하는가?
- 모든 요구사항이 구현되었는가?
- 엣지 케이스를 처리하는가?
- 에러 처리가 적절한가?

**체크리스트:**
- [ ] 요구사항과 일치
- [ ] 엣지 케이스 처리
- [ ] 에러 처리 완료
- [ ] 테스트 커버리지 충분

### 2. 코드 품질 (Code Quality)

**SOLID 원칙:**
- **S**ingle Responsibility: 한 클래스는 한 가지 책임만
- **O**pen/Closed: 확장에는 열려있고 수정에는 닫혀있음
- **L**iskov Substitution: 상속 관계가 올바름
- **I**nterface Segregation: 인터페이스가 작고 구체적
- **D**ependency Inversion: 추상화에 의존

**체크리스트:**
- [ ] 함수/클래스가 한 가지 책임만 함
- [ ] 중복 코드가 없음 (DRY)
- [ ] 불필요한 코드가 없음 (YAGNI)
- [ ] 변수/함수 이름이 명확함
- [ ] 복잡도가 적절함 (Cyclomatic Complexity < 10)

### 3. 보안 (Security)

**일반적인 취약점:**
- SQL Injection
- XSS (Cross-Site Scripting)
- CSRF (Cross-Site Request Forgery)
- 인증/인가 문제
- 민감한 정보 노출

**체크리스트:**
- [ ] 입력 검증 수행
- [ ] 출력 이스케이프 처리
- [ ] 인증/인가 확인
- [ ] 민감한 정보 보호
- [ ] 의존성 취약점 확인

### 4. 성능 (Performance)

**확인 사항:**
- 불필요한 루프가 있는가?
- 데이터베이스 쿼리가 최적화되어 있는가?
- 메모리 누수가 없는가?
- 캐싱이 적절히 사용되는가?

**체크리스트:**
- [ ] 알고리즘 복잡도 적절 (O(n) 이하)
- [ ] 데이터베이스 쿼리 최적화
- [ ] 불필요한 API 호출 없음
- [ ] 메모리 누수 없음

### 5. 테스트 (Testing)

**확인 사항:**
- 테스트 커버리지가 충분한가? (80% 이상)
- 테스트가 명확한가?
- 엣지 케이스 테스트가 있는가?
- 테스트가 독립적인가?

**체크리스트:**
- [ ] 커버리지 80% 이상
- [ ] 테스트 이름이 명확
- [ ] 엣지 케이스 테스트 포함
- [ ] 테스트 간 의존성 없음
- [ ] 모든 테스트 통과

## 리뷰 프로세스

### 1단계: 변경 사항 이해

1. 코드 설명 읽기
2. 관련 이슈 확인
3. 변경된 파일 목록 확인

### 2단계: 코드 검토

1. 전체 흐름 파악
2. 각 함수/클래스 검토
3. 테스트 코드 검토
4. 문서 검토

### 3단계: 피드백 작성

1. 긍정적인 피드백 먼저
2. 개선 사항 제시
3. 질문하기 (지시하지 않기)
4. 명확한 예제 제공

### 4단계: 의사결정

```
✅ 승인: 모든 기준 충족
⚠️ 조건부 승인: 경미한 개선 필요
❌ 반려: 주요 문제 있음
```

## 피드백 작성 가이드

### 좋은 피드백

```
// ✅ 구체적이고 건설적
"이 함수는 너무 많은 책임을 가지고 있습니다. 
검증 로직을 별도의 함수로 분리하면 테스트하기 쉬워질 것 같습니다.

예:
function validateUser(user) {
  // 검증 로직
}

function createUser(user) {
  validateUser(user);
  // 생성 로직
}"
```

### 나쁜 피드백

```
// ❌ 모호하고 비판적
"이 코드는 좋지 않습니다."
"왜 이렇게 했나요?"
```

## 리뷰 결과 형식

```
✅ 승인 / ⚠️ 조건부 승인 / ❌ 반려

평가:
- 기능 정확성: ✅ 완벽 / ⚠️ 개선 필요 / ❌ 문제 있음
- 코드 품질: ✅ 우수 / ⚠️ 개선 필요 / ❌ 문제 있음
- 보안: ✅ 안전 / ⚠️ 경미한 문제 / ❌ 심각한 문제
- 성능: ✅ 최적화됨 / ⚠️ 개선 가능 / ❌ 문제 있음
- 테스트: ✅ 충분 / ⚠️ 부족 / ❌ 없음

주요 의견:
- [의견 1]
- [의견 2]

개선 사항:
- [개선 1]
- [개선 2]

다음 단계:
Librarian 에이전트가 문서화를 진행합니다.
```

## 체크리스트

리뷰 완료 후 다음을 확인합니다:

**필수 항목:**
- [ ] 코드가 요구사항을 충족하는가?
- [ ] 테스트가 충분한가?
- [ ] 보안 문제가 없는가?
- [ ] 성능 문제가 없는가?

**권장 항목:**
- [ ] 코드 스타일이 일관적인가?
- [ ] 문서가 최신인가?
- [ ] 의존성이 최소화되어 있는가?
- [ ] 에러 처리가 적절한가?

## 주의사항

- 건설적이고 존중하는 태도로 피드백을 제공합니다
- 개인 공격이 아닌 코드에 대해 피드백합니다
- 개선 방안을 함께 제시합니다
- 과도한 완벽주의는 피합니다
