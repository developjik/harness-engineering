# AI Harness Engineering Design

본 문서는 `harness-engineering`의 목표 구조와 2026-03-12 기준 현재 구현 범위를 함께 설명합니다.

## 1. 핵심 철학

- **PDCA 기반 워크플로우**: Plan, Do, Check, Act 흐름으로 작업을 분리합니다.
- **컨텍스트 엔지니어링**: 단순 지시문이 아니라 에이전트, 스킬, 훅을 함께 제공해 작업 맥락을 강화합니다.
- **역할 분리**: 설계, 구현, 리뷰, 문서화를 서로 다른 에이전트에 맡깁니다.

## 2. 현재 구현된 것

현재 저장소는 다음 계층을 실제로 제공합니다.

### 플러그인 계층

- `.claude-plugin/plugin.json`
- 에이전트/스킬/훅 경로 등록

### 에이전트 계층

- `architect`
- `engineer`
- `guardian`
- `librarian`

### 스킬 계층

- `brainstorm`
- `implement`
- `review`
- `document`

### 훅 계층

- 세션 시작/종료 로깅
- 사용자 입력 길이 기록
- 위험한 Bash 명령 차단
- 편집 전 백업
- 편집 후 변경 추적과 선택적 린트
- 현재 에이전트 상태 기록

## 3. 현재 구현되지 않은 것

초기 설계 문맥에서 언급되었지만 아직 저장소에 포함되지 않은 항목은 다음과 같습니다.

- `ultrawork` 스타일의 통합 실행기
- 자동 PDCA 단계 전환 엔진
- 별도 세션 상태 관리자
- LLM API 통합 레이어
- 독립 실행형 CLI 또는 웹 UI

## 4. 시스템 관점에서 본 현재 구조

```text
사용자 입력
  -> Claude Code 플러그인
    -> agent prompt 선택
    -> skill 지침 활용
    -> hook 스크립트로 로그/백업/검사 수행
```

중요한 점은 현재 구조가 "에이전트 실행을 자동 오케스트레이션하는 엔진"이 아니라, "Claude Code가 읽어 사용할 수 있는 리소스 세트"라는 것입니다.

## 5. 목표 워크플로우

목표 워크플로우는 여전히 다음 PDCA 흐름입니다.

1. `Architect`: 요구사항 분석과 설계 문서 작성
2. `Engineer`: 설계 기반 구현과 테스트
3. `Guardian`: 코드 리뷰와 품질 검증
4. `Librarian`: 문서 정리와 운영 지식 축적

현재는 이 흐름을 사용자가 직접 진행해야 하며, 훅은 그 과정을 보조합니다.

## 6. 다음 로드맵

### 단기

- 로그/상태 저장 위치 통일
- 훅 입력 포맷 문서화
- 에이전트와 스킬 문서 일관성 개선

### 중기

- 자동 단계 전환을 위한 상태 머신 설계
- 통합 실행 명령 또는 오케스트레이터 추가
- 훅 테스트 시나리오 보강

### 장기

- 팀 단위 운영을 위한 템플릿 확장
- 더 세밀한 품질 게이트 추가
- 다른 개발 환경으로 이식 가능한 패키징
